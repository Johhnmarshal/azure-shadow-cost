# workbooks/

Azure Workbook templates (JSON) generated by the web app for offline use
in the Azure Portal.

The web app **generates** these on demand — they're not committed pre-built
because the queries are tied to Log Analytics custom-table names that may
need adjustment per tenant. A reference copy of the hidden-waste template
lives at [`samples/workbooks/azshc-hidden-waste.json`](../samples/workbooks/azshc-hidden-waste.json).

## Templates

| Name | Purpose | Custom table read |
|---|---|---|
| `hidden-waste` | Headline tile, by-category bars, top-25 individual offenders, sub leaderboard. | `HiddenWaste_CL` |
| `peak-rightsizing` | VM inventory + Advisor-unsafe filter + per-VM verdict table. | `PeakRightsizing_CL` |
| `ri-coverage` | Coverage gap by family×region, cumulative buffer-exposure tracker. | `RICoverage_CL` |

## How to use one

1. From the running web app: open the **Downloads** tab → **Azure Workbook
   templates** → click **JSON** for the template you want.
2. Azure Portal → **Monitor → Workbooks → New → Advanced editor → Gallery
   Template**, paste the JSON, **Save**.
3. Set up the matching Log Analytics custom table by exporting the
   relevant CSV from the web app (e.g., `samples/peak-rightsizing/sample.csv`)
   and ingesting it via Diagnostic Settings → Custom Logs, or via Logic
   App / Function pipeline.

## API surface

| Method | Path | Purpose |
|---|---|---|
| GET | `/api/workbooks` | Catalogue of templates |
| GET | `/api/workbooks/{name}.json` | Download one template |

## Why no Power BI?

Workbooks are the right surface for engineers (in-Portal, Resource-Graph
aware, KQL native). Operators who want exec-friendly dashboards typically
ingest the same custom tables into Power BI — adding a PBIX export is on
the post-v1 backlog.
